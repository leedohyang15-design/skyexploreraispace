# 학습 노트 ⑬ — 말머리 성운 스크립트: 실측(ST) 시뮬레이션 & 수정 로그

> **출처**: SkyExplorer ST에서 `zoom_horsehead.py`를 **실제 실행하며 화면을 관찰**하고
> 그 결과로 수정한 전 과정 기록(사용자 제공). 내 추측 기반 결론을 **실측으로 교정**함.
> 관련 스크립트: [`scripts/zoom_horsehead.py`](../scripts/zoom_horsehead.py) (이 로그 기준 최종판),
> 백업: `scripts/zoom_horsehead_backup.py`.

---

## 1. 원본 문제 (실행 관찰)

원본 방식: `setPositionLBR`(LOS Local 프레임) + `setOrientationP(-55)` 수동 조준
→ `setPositionR` 줌인 → `setPositionL(360)` 공전.

관찰된 문제:
- **말머리가 화면 중앙이 아니라 오른쪽 아래로 치우침** (시작·줌인·공전 내내).
- 공전(`setPositionL 360`) 시 말머리가 프레임 밖으로 흐름.
- HUD 최종값: `L:360°, B:0°, R:1.94pc` — 위치는 맞으나 구도가 어긋남.

원인: `PITCH=-55` + heading 없음 조합이 말머리 기둥을 정확히 중앙에 두지 못함.
말머리 LOS Local **프레임 원점이 상징적 '기둥'과 어긋나** 있어(그래서 큰 pitch가
필요) 어떤 고정 orientation도 공전 중 중앙을 유지 못함.

## 2. 시도한 방법과 실측 결과

| # | 방법 | 관찰 결과 | 판정 |
|---|------|-----------|------|
| A | 수동 `PITCH=-52` + `setOrientationH(43)` (정지) | 말머리 기둥이 **완벽히 중앙** (근접 정지) | ✅ 조준·줌인·중앙정렬 해결 |
| B | A + `setPositionL` 공전 (heading 고정) | 공전 시작하자 말머리가 프레임 **밖으로 드리프트** (L=82°에서 사라짐) | ❌ |
| C | B + `setOrientationH(AIM_H+360)` 동반 애니 | `PITCH`와 **짐벌 결합** → 말머리가 수직으로 흔들려 위로 뜸 (L=189°) | ❌ |
| D | `setZoomPosition` 줌 락(원점) + `setPositionL` | 엔진이 재조준 자동 처리 → 말머리 **프레임 유지**. 단 원점이 기둥과 어긋나 기둥은 약간 아래 | ✅ 드리프트 근본 해결 |
| D' | 줌 타깃 오프셋 `TZ`/`TY` 로 기둥 정확히 중앙 시도 | GreatCircle 공식이 타깃을 중앙으로 보내며 **롤(회전) 유발** | ⚠️ 오프셋은 0 권장 |
| E | 줌 락 + **완전한 360° 공전** | L≈180°에서 카메라가 성운 **뒤편**으로 넘어가 **어두운 실루엣**만 보임 | ⚠️ 근접 360°의 한계 |
| F | 줌 락 + **정면 스윙**(±75°) | 정면·삼사분면(+56°) 모두 **밝게 중앙 유지**, 배경 흐름 | ✅ **최종 채택** |

핵심 인사이트: `setZoomPosition`(줌 락)은 대상에 대한 **재조준을 엔진이 내부적으로
처리**한다 (사용자 `run_orion.py`의 검증된 패턴). 이걸로 B·C의 드리프트/흔들림을 근본 해결.

## 3. 최종 방식

1. **말머리 프레임 진입 + 줌 락** — `setZoomPosition`으로 말머리를 화면 중앙에 고정
   (카메라가 어디로 가든 유지).
2. **점진 줌인** — `setZoomFov`(FOV_START 110 → FOV_CLOSE 72)로 접근 효과, 중앙 자동 유지.
3. **정면 스윙 공전** — 베이스 카메라를 `정면(0°) → +75° → −75° → 정면` 으로 크게 스윙.
   말머리는 중앙 고정, 배경만 좌우로 흐름 = "말머리를 축으로 도는" 효과.
   정면 반구 안에서만 움직여 **어두운 뒷면으로 안 넘어감**.

---

## 4. 왜 완전한 360°가 아닌가

근접 거리(≈2pc)에서 위치를 360° 돌리면 카메라가 성운 **뒤편**으로 넘어가 앞을 비추던
붉은 발광이 사라지고 **검은 실루엣**만 남는다 (L≈180°에서 실제로 확인). 그래서 밝은
앞면을 유지하는 정면 스윙으로 설계.
완전한 360°가 꼭 필요하면 스크립트의 `SWEEP` 블록을
`cam.setPositionL(360, Anim.cubic(30), track)` 한 줄로 교체하면 됨 (뒷면 통과 감안).

## 5. 튜닝 파라미터 (스크립트 상단)

| 파라미터 | 의미 | 조정 |
|---------|------|------|
| `FOV_CLOSE` | 줌인 배율 | ↓ = 더 크게 (72→60). 기둥이 세로로 낮으면 ↑ |
| `SWEEP_DEG` | 좌우 스윙 폭 | ↑ = 배경 더 흐름. 90 근처부터 뒷면(어두움) 주의 |
| `SWEEP_TIME` | 스윙 속도 | 한 방향 시간(왕복 구간 자동 2배) |
| `R_ORBIT` | 공전 반경 | 배경 흐름 폭 |
| `TX/TY/TZ` | 줌 타깃 오프셋 | **0 권장** (건드리면 롤 발생) |

## 6. 남은 미세 이슈

- 말머리가 세로로 **살짝 아래**에 위치 (롤 방지를 위해 줌 타깃을 원점에 둔 결과).
  더 올리려면 `FOV_CLOSE`를 키우거나 별도 보정 필요.
- 백업: 원본은 `zoom_horsehead_backup.py` 로 보관.

## 7. 내 이전 결론 교정 (중요)

| 내가 이전에 내린 (틀린) 결론 | 실측(ST) 교정 |
|------|------|
| "`setZoomPosition`(줌 락)은 방향이 잠겨 회전 불가 → FadeTo로 가야" | ❌ 정반대. 줌 락은 **엔진 자동 재조준**으로 공전 중 중앙 유지 = **드리프트의 정답**(D/F). |
| "회전은 `setOrientationR`(롤)/`adv.roll()`로" | 롤은 제자리 핀휠 회전이라 "축으로 돈다"가 아님. **베이스 위치 L 스윙**(줌 락 위에서)이 정답. |
| "수동 pitch로 중앙정렬하면 됨" | 정지엔 OK(A)지만 **공전 시 드리프트**(B), heading 동반은 **짐벌 흔들림**(C). |
| "완전한 360° 공전" | 근접에선 **뒷면(검은 실루엣)** 통과 → **정면 스윙 ±75°**(F)가 실용 정답. |

→ 교훈: **화면 관찰 없는 기하 추론은 자주 틀린다.** "천체 중앙 고정+공전"은
`setZoomPosition` 줌 락(엔진 재조준) 위에서 **베이스 위치를 움직이는** 것이 정석
(별=`run_orion.py`, 성운=이 로그 둘 다 검증).
