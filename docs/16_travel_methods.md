## 0. 한눈에 보기 — 이동 방법 선택표

| 방법 | 화면 양상 | 도착 상태 | 용도 |
|---|---|---|---|
| **GoTo** | ✅ 진짜 연속 비행 (20초 표준) | 대상별 도킹 지점 (아래 표) | 우주선 여행 연출의 정답 |
| **FadeTo** | Fade out→순간이동→Fade in | R≈5 트랙반지름, Target 30 | 빠른 장면 전환 |
| **StraightGoTo / GoToPlace** | 순간이동 | — | 즉시 배치 |
| **ConnectTo** | 이동 없음 (프레임만 전환) | R 단위가 대상 반지름으로 | 프레임 전환 (GoTo 와 병용 금지!) |
| **Land (버튼/재현)** | ✅ 하강+몸 일으키기 10초 | 지표 R=0, SkyView | 착륙 연출 (GoTo 지구 후) |
| **같은 프레임 R/L 애니** | ✅ 연속 이동 | — | 줌인/풀백/오빗/표류 |

## 1. GoTo 내부 안무 — 대상 종류별로 다르다! (Recording5 완전 해독)

공통 전주: `Time stop → Camera stop → (273/295 클리어) → Turn ON bodies → Switch(트랙 전환)`
— **비행 전에 목적지 트랙으로 전환**되고, 273(위치)+295(방향) 애니메이션이 비행의 실체.

### ① 행성 GoTo (예: 토성) — 1단 비행
```
Go to: 273 위치 → {L=60, B=20, R=5}   (20초)
       295 방향 → {-120, -20, 0}      (6.7초, 비행 중)   ← '비행 중 회전'의 정체 (제거 불가)
Set ori to XYZR → 295 {0,0,0,0} → Switch to final port (EquSync)
```
- **도킹: R=5 행성반지름** (FadeTo 와 동일 프레이밍), target={0 30} 포함.
- 비행 시간 20초 고정 (우리 도착 감지 17초와 일치 — 감지가 살짝 이름).

### ② 지구(관측지 행성) GoTo — 도킹 후 '착륙은 별도'
```
(공통 전주) + Set place2D position ← ⚠️ 엔진이 Place2D 를 임의값으로 덮어씀!
   (Recording4: 위도20/경도-90, Recording5: 위도20/경도106.5 — 그때그때 다름, 아마 직하점)
Go to: 273 → {0, 89.999, 4}   (20초) — 북극 상공 R=4 지구반지름
       295 → {-180, -89.999}  (6.7초) — 수직 하방 응시(지구를 내려다봄)
Switch to final port → Make sure manipulator can handle: 295 {180, -90} 즉시
→ 여기서 끝! R=4 '호버' — R=0 자동 착륙 아님!!
```
- ⚠️ **미리 설정한 관측지 좌표는 소용없음** — 착륙 후 딥(암전)에서 재지정이 정석.
- ⚠️ 출발 전 setDateTime 큰 점프 = 출발지 하늘 통째 회전 (v16 사고) — 시각 변경도 착륙 후로.

### ③ 은하수(은하) GoTo — 3단 안무 (신규 해독!)
```
(공통 전주)
Move away : 276 R → 77.4e9 (13.3초)  ← 지구에서 먼저 '후진 이탈'!
Look at   : 295 → {228.9, -67.2, 0, 4.9e20} (13.3초, 이탈 중 은하수 조준)
Switch position and orientation      ← 프레임 점프 (지구 트랙 → StdGal 은하 트랙)
Go to     : 273 → {180, 20, 16.8} (15.3초) + 295 {0,-20,0} — 은하 프레임 안 비행
```
- **'Move away + Look at' 페이즈 덕분에 은하수 GoTo 가 제일 비행답게 보였던 것.**
- 도킹 R=16.8 (StdGal 단위, kpc 추정). 총 ~35초 (우리 도착 감지 35초 일치).

## 2. Land — 착륙은 GoTo 가 아니라 별도 명령 (Recording4/5 해독, v22 재현 성공)

지구 GoTo 도킹(R=4 호버) 후, UI Land 버튼의 실체 = **3연발 동시 10초**:
```python
hp = cam.positionLBR                      # 호버 위치 읽기 (L≈0, B≈90, R≈4)
cam.setPositionLBR(Vec(hp.x, hp.y, 0.0), Anim.cubic(10.0), -1)  # ① R→0 하강
h = cam.orientationHPR.x                  # 현재 헤딩 (녹화값 ≈180)
cam.setOrientationHPR(Vec(h, 0.0, 0.0), Anim.cubic(10.0))       # ② pitch -90→0 몸 일으키기
# ③ (원본은 setTarget 보정 — 우린 착지 후 setTargetHeight(30) 으로 대체)
```
- 착지 후 `state=SkyView, CanTakeOff, R=0` — 지상 복귀 완료. reset 불필요!
- 텔레메트리 실측: R 4.01 → 3.59 → 2.60 → 1.41 → 0.42 → 0.000 (10초 cubic).

## 3. ConnectTo — 이동 없는 프레임 전환 (v13 텔레메트리)

- 트리거 즉시 카메라 좌표가 대상 트랙 기준으로 재해석됨 (토성 이탈 후 지구 ConnectTo →
  R 60(토성반지름) → 220529(지구반지름) = 14.05억km = 토성↔지구 거리 정확 일치).
- ⚠️ **ConnectTo 상태에선 GoTo 비행이 동결** (60초 R 불변) — 같이 쓰지 말 것.
- ConnectTo + 수동 R 하강(다이브 연출 시도) = 느린 드리프트만, 실패 → **프레임 간
  '보이는 다이브'는 미해결. 정답은 GoTo 도킹 + Land 재현.**

## 4. 같은 프레임 안 연속 이동 — 유일하게 우리가 완전 제어하는 비행

도킹/FadeTo 후 그 프레임 안에서는 `setPositionR(읽은값×배율, Anim, -1)` 이 진짜 연속 이동:
- 줌인: `R×0.5` (토성 고리 클로즈업, 6초 cubic — 화면 중앙 고정)
- 풀백: `R×12` (이탈 후진 비행, 8초)
- 표류: `R×1.5` (은하수 도착 후 서서히 멀어짐, 10초)
- ⚠️ 절대값 금지 (R 단위 = 트랙 대상 반지름), track=-1 (현 프레임 유지) 필수.
