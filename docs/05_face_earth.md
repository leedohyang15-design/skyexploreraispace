# 학습 노트 ⑤ — 지구 정면 보기 (직접 설계, 정답 없음)

> 과제: 토성 예제처럼 **지구를 정면으로 본** 스크립트를 직접 설계. 정답지 없음.
> - 구현: [`scripts/my_face_earth.py`](../scripts/my_face_earth.py)

## 설계 결정과 근거

토성 예제(`my_freefly_saturn.py`)의 **저수준 카메라 배치** 패턴을 재사용하되, 대상과 구도값만 지구 정면에 맞게 바꿨다.

| 항목 | 선택값 | 이유 |
|------|--------|------|
| 좌표계 | `Earth` `PlanetPort.EquatorialSynchronous` | 적도 동기면 → 고정된 한 면이 정면. 토성 예제와 동일 계열 |
| L (경도) | `0°` | 본초자오선이 정면(대략 아프리카/유럽 방향) |
| B (위도) | `0°` | 적도면에서 수평으로 봄 = **"정면"** (토성은 B=20°로 위에서 비스듬) |
| R (거리) | `30000 km` | 지구 반지름(≈6371km)의 약 4.7배 → 화면에 큰 원반 |
| target | `Vec2(0, 0)` | 정면 수평 응시 (토성의 0/30 대신 0/0) |

## 토성 예제와의 차이 (의도적 변경점)

- **B: 20° → 0°** : 위에서 내려다보는 비스듬한 뷰가 아니라, 적도 정면 수평 뷰.
- **target height: 30 → 0** : 시선을 수평으로 맞춰 지구를 화면 중앙 정면에 둠.
- **R: 701950 km → 30000 km** : 지구가 토성보다 훨씬 작으므로 거리도 비례해 축소.
- **대상**: `Planet.PlanetName.Saturn` → `Earth`, 나머지 골격(reset→암전→배치→intensity→페이드인→FreeFly)은 동일.

## ⭐ 실행으로 발견한 핵심 버그: R 단위 = 행성 반지름 (km 아님)

첫 실행 결과: 지구가 돔 하단에 **점**으로만 보이고 HUD `R: 1.28 AU`.
원인 분석:

```
입력 R = 30000  ×  지구 반지름 6371 km  =  1.2776 AU  ≈  HUD의 1.28 AU
```

→ **행성 기준 좌표계에서 `setPositionLBR`의 R 단위는 km가 아니라 "행성 반지름"**.
사진 HUD가 `km`로 표시되길래 그 숫자(30000)를 그대로 입력한 게 잘못이었다.
- 수정: `R = 30000` → **`R = 5`** (지구 반지름의 5배 ≈ 31,855 km).
- 같은 실수가 토성 스크립트에도 있었음: `R=701950`(km로 착각) → 토성반지름 ≈58232km의
  약 12배이므로 **`R=12`** 가 정답. (`my_freefly_saturn.py` 동시 수정)

### 좌표계별 R 단위가 다르다 (중요)
| 기준 좌표계 | R 단위 | 근거 |
|-------------|--------|------|
| 행성(Planet) port | **행성 반지름** | Earth: 30000 입력 → 30000×6371km = 1.28 AU (실측) |
| 태양/별(IndividualStar Ecliptic) | **AU** | FreeFlySun: R=30 입력 → HUD 30.00 AU |

→ "천체 가까이서 보기"는 보통 **반지름의 몇 배(한 자리 수)**, 태양계 전체 조망은 **AU**.

### "천체가 화면 하단" 은 버그 아님
태양/토성 예제 이미지에서도 천체는 돔 **하단-중앙**에 위치했다. 즉 이 구도(target 0/0~0/30)의
정상 결과이며, R만 적정값이면 지구가 하단-중앙에 큰 원반으로 보인다.

## ⭐ 2차 실행: 거리는 맞았지만 "겨냥(aim)"이 문제 → FadeTo로 전환

R=5 수정 후 실행: 지구가 큰 구로 보였으나 **돔 하단에 걸쳐 윗부분만** 보임(잘림).
HUD `R: 31891 km`(=5×지구반지름)로 거리는 정확. 남은 문제는 **카메라 겨냥/구도**.

- 저수준 `setTarget(azimuth, height)` ↔ 어안(fulldome) 화면 위치 매핑은 프레임 기준축 +
  비선형 투영 때문에 **런타임 반복 없이는 정확히 못 맞춤**.
- 태양/토성 예제도 천체를 **하단**에 두는 establishing-shot 구도라, 거기서 정중앙으로
  옮기는 각도는 추정값이 된다.
- **결론: "구형 전체가 깔끔히 보이게"는 고수준 `FadeTo`가 정답.** 엔진이 대상에 맞는
  거리·겨냥을 자동 계산 → 잘림 없이 프레이밍. (노트 ④ 교훈 재확인: 정확좌표=저수준,
  깔끔한 천체뷰=고수준.) `my_face_earth.py`를 FadeTo 방식으로 교체함.
- 미세조정이 필요하면 FadeTo 후 `Camera.setPositionR(value, Anim, track)`로 거리만 조절
  (value 단위=행성 반지름, 4~6 권장; 겨냥은 FadeTo가 잡아둔 상태 유지).

## ⭐ 3차: FadeTo + 부분 보정으로 "중앙·크게" (최종 패턴)

FadeTo 결과(L=9.52°, B=90°, R≈4 지구반지름, target 30°)는 지구가 **하단-중앙에 작게**.
요구: 더 중앙 + 돔 절반 가까이 크게. → **FadeTo를 통째로 덮지 않고 두 속성만 보정**:

| 목표 | 방법 | 비고 |
|------|------|------|
| 더 크게 | `Camera.setPositionR(value, Anim, track)` | value=지구반지름 단위. 4.0→1.3 (돔 ~42%) |
| 더 중앙 | `Camera.setTargetHeight(0.0, Anim)` | 시선 고도↓ → 천체가 화면 위로. (방위각은 FadeTo 유지) |

- **핵심 기법**: `setTarget`(az+height 동시)으로 FadeTo 겨냥을 날리지 말고, **`setPositionR`(거리만)·`setTargetHeight`(고도만)** 단일 속성 세터로 보정 → FadeTo의 좋은 구도를 보존하면서 원하는 축만 조정.
- **크기 물리 한계**: 180° 풀돔에서 지구가 화면 50% 이상 차려면 R≈1.0(지표면 근접)이라 사실상 Land. 실용 최대는 R≈1.1(~48%). 그래서 R=1.3(~42%)을 큰-안전값으로 선택.
- **겨냥 방향 가설(검증됨에 가까움)**: "천체가 중심 아래로 벗어난 각 ≈ target height". 그래서 height 30→0이면 천체가 중앙으로 올라옴. (틀리면 height를 60으로 키워 반대 보정.)

## ⭐ 4차: 실행 피드백으로 두 가지 교정 (중요)

3차 시도(`setPositionR(1.3)` + `setTargetHeight(0)`) 실행 결과:
- HUD `R: 25513 km` **변화 없음** → `setPositionR` 가 적용 안 됨.
- 지구가 **더 아래로** 내려감.

여기서 확정된 두 사실:

1. **setTargetHeight 방향 (가설 뒤집힘)**: 값을 **낮추면 천체가 화면 아래로** 간다.
   → 중앙으로 올리려면 **고도를 높여야** 한다 (30 → 60). (앞서 "낮추면 중앙"은 틀림.)
2. **FadeTo 후 `setPositionR`(planet port) 무효**: FadeTo 가 카메라를 **Place2D(Earth)** 에
   연결(`Place2D 1 Earth` HUD 표시)하기 때문에, 행성 포트 기준 거리 세터가 무시됨.
   → 거리 제어는 `setPositionLBR`(L,B,R 전체 지정, R=지구반지름)로 하되 FadeTo 의 방향값
   (L≈14°, B=90°)을 그대로 넣어 구도를 유지한다.

**최종 패턴**: FadeTo(방위각 자동) → `setPositionLBR(Vec3(14,90,1.6), …)`(거리 제어)
→ `setTargetHeight(60)`(중앙 정렬) → FreeFly.

> 일반화: FadeTo 는 카메라를 대상의 Place2D 에 바인딩하므로, 이후 *부분 거리 세터
> (setPositionR)* 가 안 먹을 수 있다. 거리까지 바꾸려면 **전체 위치(setPositionLBR)** 로
> 같은 방향을 다시 지정하는 게 안전하다.

## ⭐ 5차: FadeTo 바인딩 확정 → 완전 수동으로 전환 (결정적)

4차(setPositionLBR 1.6 + setTargetHeight 60) 실행 결과: **HUD R=25513km(4반지름), Target=30°
그대로** — 내 두 명령 모두 무시됨. 3차와 사실상 동일한 화면.

→ **확정: FadeTo 는 카메라를 `Place2D Earth` 에 바인딩하며, 이후의 카메라 거리/타깃
명령을 무시하고 FadeTo 의 상태로 되돌린다.** "FadeTo + 부분보정" 전략은 이 바인딩 때문에
원천적으로 불가.

**해법 = FadeTo 제거, 완전 수동 제어** (초기 수동 실험에서 R 제어가 검증됨):
- **B=90 극축 정렬**이 핵심 트릭: 카메라를 지구 극축 위에 두면 target 의 azimuth 가
  무의미 → 그동안 못 맞춘 '겨냥'의 불확실성이 사라지고, **height 한 값으로 상하만** 조절.
- 노브: `R`(크기, 작을수록 큼) / `target height`(상하, 높을수록 위로).

| 시도 | 방식 | 결과 |
|------|------|------|
| 1~2차 | 수동, B=0, R 단위 착오 | R 단위(반지름) 발견 |
| 3차 | FadeTo + setPositionR/Height | setPositionR 무시, height는 적용(방향 반대 확인) |
| 4차 | FadeTo + setPositionLBR/Height | **둘 다 무시**(바인딩 확정) |
| 5차 | **완전 수동, B=90, R=1.6, height=60** | 거리·상하 모두 직접 제어 가능 |

## ⭐ 6차: 수동 제어 성공 → 풀돔 기하로 중앙 정렬

5차(완전 수동) 실행: **HUD R=10205km(=1.6반지름), Target=60° 적용 성공.** 거리·고도 직접
제어 확립. 지구가 크게 나왔으나 화면 하단(~67%)에 치우쳐 잘림.

**풀돔 투영 원리로 해결**: *시선축(target 방향)이 향하는 곳 = 화면 정중앙.*
- 좌우: 지구가 이미 좌우 중앙 → `azimuth=0` 정확.
- 상하: 시선이 지구보다 60° 위를 봐서 지구가 ~60° 아래(67%)로 보임.
  → `height` 를 0 으로 낮추면 시선축이 지구를 정조준 → **상하 중앙**.
  (검증: height ↑ → 지구 ↓, 즉 height 와 화면상 천체 위치는 반대. `오프셋 ≈ target height`.)
- **TraceMode 는 추적이 아님**(장노출 별궤적 효과) — 정조준 용도로 못 씀.

## ⭐ 7차: 방향 반전 확정 — height ↑ 일수록 천체 ↑

6차(height=0) 실행: 지구가 **바닥 실오라기**로 더 내려감. → 내 가설(낮출수록 중앙)이 반대.

**실측 2점으로 관계 확정 (B=90):**
| target height | 지구 화면 위치(위=0%, 아래=100%) |
|---------------|----------------------------------|
| 60 | ~67% (아래쪽, 큼) |
| 0 | ~93% (바닥) |

→ **화면 위치% ≈ 93 − 0.43 × height.** height 가 **클수록 천체가 위로**.
**정중앙(50%)은 height ≈ 90~100.** ("시선축=중앙" 모델과 부호 반대 — 돔 투영이 상하 반전.)

**최종 중앙 정렬 (B=90)**: `target azimuth=0, height≈90`(천정) → 지구 정중앙.
크기는 `R`(반지름 단위, 작을수록 큼)로 독립 조절. R=1.3 ≈ 돔 42%.
미세조정: 아직 아래면 height 95~100, 너무 위면 80~85.

## ⭐ 8차: 적도를 화면 중앙에 — B(카메라 위도)로 시점 선택

7차에서 지구가 중앙·크게 성공. 다만 B=90 이라 **극을 정면**으로 봄(적도는 원반 테두리).
요구: **적도가 원반 중심을 가로지르게**.

- **B = 카메라가 천체를 보는 위도**: `90`=극 정면, `0`=적도면에서 옆으로(적도가 중심 수평선).
- **B(시점 위도) 와 target height(화면 상하 정렬)는 독립**:
  height=0 이면 B에 상관없이 천체가 바닥 → 정렬은 height 가, 보는 면은 B 가 담당.
- → **B 만 90→0** 으로. height=90(중앙)·R(크기)은 그대로 유지.

### 지구뷰 제어 (주의: B 와 정렬은 독립 아님)
| 원하는 것 | 노브 | 값 |
|-----------|------|-----|
| 좌우 중앙 | target azimuth | 0 |
| 상하 중앙 | target height | ≈90 (높을수록 위) |
| 크기 | R | 작을수록 큼 (1.3≈돔42%, 3≈20%) |
| 보는 경도(어느 면) | L | 도 단위 |

## ⭐ 9차: B 변경이 정렬을 깸 → 정중앙은 B=90 고정

8차에서 "적도 중앙"을 위해 B=0 으로 바꿨더니 **지구가 좌상단으로 치우쳐 잘림**.
→ **B 는 target height 와 독립이 아니다**(8차의 독립 가정은 오류). B 를 바꾸면 정렬이 깨진다.

**정중앙 조합(수정): `B=90, target azimuth=0, target height≈110`.**
- ⚠️ **height 90 ≠ 정중앙**: 실측 결과 height 60→90 에서 지구가 거의 안 움직이고(67%→68%)
  화면 65~68%(아래쪽)에 머묾 → height 가 90 부근에서 포화. 진짜 중앙(50%)엔 **height 를
  90 너머(≈110~130)**로 더 올려야 함. (7차에서 "중앙"으로 보인 건 R 이 작아 크게 차서 그렇게 보인 것.)
- **height ↔ 화면위치는 비선형**(저각에서 민감, 90 부근 포화). 정밀 정렬은 런타임에서 ±20씩 튜닝.
- 크기만 R 로 독립 조절(정렬 안 깨짐). 레퍼런스처럼 작게는 R=3.
- 별/은하수 배경: `Stars(StarrySky).setIntensity(1)`, `Galaxy(MilkyWay).setIntensity(1)`.
- 적도를 중앙선으로 보고 싶으면 B=0 으로 가되, 그 프레임에서 정렬 height 를 **다시** 튜닝해야
  함(B마다 정렬값이 다름). 정중앙 우선이면 B=90 유지가 안전.

## (구버전) 저수준 수동 조정 포인트 — 참고용

- 지구가 작으면 `R`↓(예: 4), 너무 크거나 잘리면 `R`↑(예: 7~8).
- 다른 면(아메리카/아시아)을 정면에: `L` 변경(미주 ≈ 270°, 아시아 ≈ 90°).
- 약간 위에서 보려면 `B`를 10~20°로.
- 지구를 화면 더 중앙으로 올리려면 `target` height를 약간 키움(예: `Vec2(0, 10)`).
- "정면 그대로 정지"엔 `EquatorialSynchronous`가 적합. 천구 적도 고정은 `Equatorial`/`EquatorialJ2000`.

## 고수준 대안 (참고)

단순히 "지구로 이동"만 원하면 토성 정답처럼 한 줄로도 가능:
```python
DataManager.database().data(Data.Type.PlanetType, "Earth").action(Action.Type.FadeTo).trigger()
```
단, 이 경우 정면/거리 구도는 FadeTo 기본값을 따르므로, **정확한 정면 구도가 필요해 저수준 방식을 택함**.
